Node.js Install and Setup

Install

There are many way to insall Node.js. You can download it directly from Node.js website

I recommend you to use nvm (Node Version Manager) to install Node.js
Windows user here -> nvm-windows

The pros of using nvm is you have better control and understaning of Node.js version and which version you are using locally.

You can find the full documentation on the Github page. I would like to highlight some important commands here. 


            // Install Node.js version 4.4.7 (LTS version)
            nvm install 4.4.7

            // Switch to Node.js 4.4.7
            nvm use 4.4.7

            // Basically this is all you need to install Node.js.
            // Here are some extra commands for fun

            // List all your local versions of Node.js
            nvm ls

            // List all available versions of Node.js
            nvm ls-remote

Hello World in Node.js

First you create a file named "hello.js" and paste the following code 


            console.log('Hello Wolrd!');

Then you type the following command in terminal 


            node hello.js

Here we go~

Install Packages

We use npm (node package manager) to insall Node.js packages. npm comes with Node.js so you don't have to install npm separately.

For example, if we want to install express 


            npm install express

We can also use package.json to manage dependencies. To create a package.json, type 


            npm init

Then you would get something like this 


            {
              "name": "cse135-demo",
              "version": "1.0.0",
              "description": "A demo package.json file for CSE 135",
              "main": ".mongorc.js",
              "dependencies": {},
              "devDependencies": {},
              "scripts": {
                "test": "echo \"Error: no test specified\" && exit 1"
              },
              "author": "Martin Gao",
              "license": "ISC"
            }

The next time, if we install express with "--save", express would be written into package.json file (in dependencies) 


            npm install express --save
          

            {
              "name": "cse135-demo",
              "version": "1.0.0",
              "description": "A demo package.json file for CSE 135",
              "main": ".mongorc.js",
              "dependencies": {
                "express": "^4.14.0"
              },
              "devDependencies": {},
              "scripts": {
                "test": "echo \"Error: no test specified\" && exit 1"
              },
              "author": "Martin Gao",
              "license": "ISC"
            }

"express": "^4.14.0" means "Compatible with version 4.14.4". More info about package.json verion syntax

So the next time, when you or others want to install your project, they don't have to manually install every package.
Instead, just type in "npm install", and npm will read package.json and install all the dependencies from it. 


            npm install

Tips

Don't push your Node.js libraries to Github! They are HUGE! Could be 100+MB!
Always use .gitignore to ignore those libraries.

Create a file named ".gitignore" in the root dir of your project and paste the following code into it 


            node_modules/

So that your git will ignore anything inside "node_modules".
